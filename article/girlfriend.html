Vulnhub靶场之Me-and-my-girlfriend

Vulnhub 渗透测试 CTF
完整的Vulnhub靶场"Me-and-my-girlfriend"渗透测试过程，从信息收集到提权的详细步骤。

下载连接: https://download.vulnhub.com/meandmygirlfriend/Me-and-My-Girlfriend-1.ova

前置工作

下载完成后，我们需要将靶场环境导入到虚拟机软件中进行测试。

导入虚拟机

双击下载好的 .ova 文件

在弹出的对话框中选择导入到 VMware

网络配置

为了确保测试环境的安全性，我们需要为这个虚拟机配置仅主机网络模式。这样可以：

  • 避免靶机与外网连接
  • 确保攻击机和靶机在同一网段内
  • 防止对真实网络造成影响

渗透测试

由于我们对这个靶场没有任何预先信息，因此需要从零开始进行全面的渗透测试。按照标准渗透测试流程，我们将依次进行：信息收集、漏洞发现、漏洞利用、获取访问、权限提升。

信息收集

信息收集是渗透测试的第一步，也是至关重要的一步。通过全面的信息收集，我们可以了解目标系统的基本情况，为后续攻击奠定基础。

准备工作

首先，将外网网卡断开连接，避免网络冲突和潜在的安全风险。确保攻击机和靶机在同一网段内。

主机发现

已知攻击机与靶机在同一网段（192.168.19.x），我们可以使用以下两种方法进行内网主机发现：

方法一：使用 arp-scan 进行ARP扫描

arp-scan 通过发送ARP请求来发现同一网段内的主机，速度更快、更准确。

arp-scan -l

方法二：使用 nmap 进行网络扫描

nmap 是功能强大的网络扫描工具，可以扫描整个网段的主机。

nmap 192.168.19.0/24

扫描结果分析

通过扫描，我们成功定位到目标主机：

  • 目标主机IP：192.168.19.129
  • 开放端口：22（SSH）、80（HTTP）

端口服务分析

  • 端口 22：SSH服务，可能用于远程登录，是后续获取Shell的重要入口
  • 端口 80：HTTP Web服务，通常存在Web应用，是渗透测试的重点目标

下一步我们将重点针对80端口的Web应用进行深入分析。

访问端口

在确认了目标IP和开放端口后，我们开始深入分析80端口的Web应用。

初步访问

打开浏览器，访问 http://192.168.19.129

页面显示"Only localhost can access this page"，这表明网站有IP访问限制。

源码分析

查看页面源码，了解页面结构和可能存在的线索

绕过IP限制

看到"Only localhost"的提示，我们尝试使用 X-Forwarded-For Header 绕过IP限制。

原理说明： X-Forwarded-For 是一个HTTP请求头，用于识别通过HTTP代理或负载均衡器连接到Web服务器的客户端最原始的IP地址。很多应用程序使用这个头部来判断客户端的IP，但这个头部是客户端可控的，因此可以被伪造。

操作步骤：

  1. 安装浏览器插件（如 ModHeader 或 X-Forwarded-For Header）
  2. 设置 X-Forwarded-For 头的值为 127.0.0.1（本地回环地址）

在插件中配置：

  • Header: X-Forwarded-For
  • Value: 127.0.0.1

绕过成功

刷新页面，成功绕过IP限制，显示出正常的网页内容

页面显示的是一个用户登录和注册界面，提示我们可以注册一个新用户并找到女朋友。

目录扫描

为了发现更多隐藏的文件和目录，我们使用目录扫描工具进行深入探测。

使用 dirsearch

dirsearch 是一个功能强大的目录扫描工具，可以快速发现网站上的隐藏路径。

dirsearch -u "http://192.168.19.129/?page=index"

命令说明：

  • -u：指定目标URL
  • ?page=index：指定扫描的起点页面

扫描结果分析

对扫描发现的路径逐一访问测试，在 robots.txt 文件中发现了关键信息

robots.txt 是什么？ robots.txt 是网站告诉爬虫哪些页面可以抓取、哪些不能抓取的文件。虽然它的主要作用是指导搜索引擎爬虫，但有时也会泄露一些敏感信息或隐藏路径。

总结

技术要点总结

1. 信息收集阶段

  • 使用 arp-scan 和 nmap 进行内网主机发现，定位目标IP 192.168.19.129
  • 发现开放端口：22(SSH)、80(HTTP)
  • 端口扫描是渗透测试的第一步，为后续攻击奠定基础

2. Web应用侦察

  • 通过修改 X-Forwarded-For Header 为 127.0.0.1 绕过IP访问限制
  • 使用 dirsearch 进行目录扫描，在 robots.txt 中发现关键文件 heyhoo.txt
  • 善于使用目录扫描工具和查看 robots.txt 等常见位置

3. 敏感信息获取

  • 注册并登录后发现 GET 传参漏洞：?page=profile&id=5
  • 通过修改 id 参数遍历用户信息（id=1 到 id=5）
  • 利用浏览器开发者工具修改 input 标签的 type 属性（password → text）查看明文密码
  • 最终获取到目标用户 alice 的凭据：alice / 4lic3

4. 获取初始访问权限

  • 使用获取的 SSH 凭据成功登录目标系统
  • 在用户目录 .my_secret/flag1.txt 中找到第一个 flag

5. 权限提升

  • 检查 Web 服务器配置文件（Apache 配置目录）
  • 在配置文件中发现明文的 root 用户密码：ctf_pasti_bisa
  • 使用 root 凭据通过 SSH 提权成功
  • 在 /root/flag2.txt 中找到第二个 flag

安全启示

  • 弱口令风险：系统使用了简单的密码，容易被猜测或暴力破解
  • 配置文件泄露：敏感的数据库或管理员凭据不应明文存储在配置文件中
  • 输入验证不足：GET 参数可直接遍历用户信息，缺少访问控制
  • IP过滤不可靠：仅依赖 HTTP Header 进行IP过滤容易被绕过
  • 前端验证不可信：密码字段通过前端隐藏，后端未做额外保护
